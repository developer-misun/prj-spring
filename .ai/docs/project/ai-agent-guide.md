
# AI 에이전트 협업 가이드

> 이 문서는 AI 코딩/설계 에이전트와 함께 작업할 때 지켜야 할 원칙과 주의사항을 정리합니다.  
> 모든 API 프로젝트에서 공통으로 사용하며, 필요 시 계속 업데이트합니다.

---

## 1. 기본 원칙

- AI 에이전트는 **보조 도구**이지, 최종 의사결정자가 아니다.
- 에이전트가 제안한 설계/코드/문서는 **항상 사람이 리뷰**한 후 채택한다.[web:93][web:95]
- 보안, 성능, 라이선스와 관련된 부분은 특히 더 엄격하게 검토한다.[web:90][web:93][web:96]

---

## 2. 프롬프트(요청) 작성 원칙

- **맥락을 먼저 설명**한 뒤, 구체적인 작업을 요청한다.
  - 예: 프로젝트 목적, 도메인, 사용 기술, 이미 결정된 규칙(naming 등).[web:90][web:95][web:97]
- “무엇을”, “왜”, “어디까지”를 명확하게 적는다.
  - 예: “테스트용 GET API 하나만”, “예외 처리 최소”, “Spring Boot 3 기준”.
- 한 번에 너무 많은 작업을 맡기지 않고, **작은 단위로 나눠서 요청**한다.[web:94][web:97]

> 프롬프트가 모호하면, 결과도 모호하고 불안정해질 수 있다.[web:90][web:94]

---

## 3. 설계·문서 작업 시 주의사항

- AI가 제안한 **API 설계**는 먼저 `docs/api-design` 템플릿에 맞게 사람이 정리한다.
- 네이밍/URI/JSON 규칙은 항상 `docs/naming/naming-rules-<domain>.md` 를 기준으로 검증한다.[web:75][web:88]
- 에이전트가 만들어 준 문서는:
  - 섹션 구조가 템플릿과 맞는지,
  - 용어가 도메인 언어와 일치하는지,
  - 다른 문서와 **중복·충돌**이 없는지 확인한다.[web:87][web:89]

---

## 4. 코드 생성/수정 시 주의사항

- AI가 생성한 코드는 **항상 사람 리뷰 + 정적 분석 도구**를 통과한 후에만 커밋한다.[web:90][web:93][web:95]
- 특히 다음 항목은 매번 확인한다:
  - 입력 값 검증/필터링이 빠져 있지 않은지 (SQL 인젝션 등).[web:96]
  - 불필요하게 많은 외부 라이브러리를 추가하지 않았는지.[web:96]
  - 기존 아키텍처/레이어 구조를 깨지 않았는지.[web:76][web:96]
- 보안 관련 코드(인증/인가, 암호화, 비밀키 취급)는:
  - AI가 초안을 만들 수는 있지만, 반드시 **수동 검토 + 사내 가이드**로 재검증한다.[web:90][web:96]

---

## 5. 데이터/보안 관점 주의사항

- 외부 AI 서비스에 다음을 직접 붙여넣지 않는다:[web:91][web:93][web:96][web:99]
  - 비밀키, 인증 토큰, 비밀번호
  - 고객/개인정보(PII)
  - 회사 내부 기밀(설계 문서, 비공개 알고리즘)
- 필요 시 민감한 부분은 **마스킹하거나 단순화**해서 질의한다.
- AI가 제안한 코드/설계에 **민감한 정보 하드코딩**이 없는지 반드시 확인한다.
  - 예: DB 비밀번호, API 키, 내부 URL 등.[web:93][web:96]

---

## 6. 라이선스/출처 관련 주의사항

- AI가 생성한 코드라도, 사실상 외부 코드 조각과 비슷할 수 있으므로:
  - 대형 라이브러리/프레임워크 전체를 통째로 붙여 넣은 듯한 코드는 경계한다.[web:93]
  - 서드파티 코드/스니펫을 사용할 때는 해당 라이선스를 별도로 확인한다.
- 공통 가이드:
  - “출처 불명인 대형 코드 블록”은 검토 없이 바로 사용하지 않는다.[web:93]

---

## 7. 작업 방식(루프) 가이드

- 기본 루프는 다음을 따른다:[web:95][web:97][web:98]
  1. Plan: 문제 정의/배경/목표를 짧게 정리한다.
  2. Generate: AI 에이전트에 구체적인 작업을 요청한다.
  3. Review: 결과를 검토·수정하고, 필요 시 부분만 다시 요청한다.
  4. Test: 실제 실행/테스트를 통해 검증한다.
  5. Repeat: 부족한 부분만 다시 Plan → Generate 루프를 돌린다.
- 한 세션에서 맥락이 길어지면:
  - 중간에 현재 상태를 정리한 후, 새로운 세션(대화)에서 다시 시작하는 것이 좋다.[web:95][web:97]

---

## 8. 역할 정의 (AI 에이전트 vs 사람)

- AI 에이전트:
  - 코드/문서 초안 작성
  - 설계 옵션 제안, 예시 코드 제공
  - 오류 메시지/스택 트레이스 해석 도움[web:97]
- 사람(개발자):
  - 도메인/요구사항/제약 조건 결정
  - 최종 설계/코드 승인
  - 보안/성능/운영 관점에서의 최종 책임

> 기본 원칙: **AI는 “똑똑한 초안 생성기 + 주니어 페어 프로그래머” 정도로 취급하고,  
> 최종 품질과 책임은 항상 사람이 진다.**[web:93][web:95][web:98]
