# AGENTS.md

> 이 문서는 AI 코딩 에이전트가 프로젝트를 이해하고 작업을 시작할 때 가장 먼저 참고하는 출입구 문서입니다.  
> 상세한 요구사항, 규칙, 설계 문서, 작업 계획은 `.ai/` 디렉터리 아래의 문서를 기준으로 확인합니다.

---

## 1. 기본 원칙

- AI 에이전트는 작업을 시작하기 전에 이 문서를 먼저 읽고 `.ai/` 디렉터리 구조를 파악합니다.
- 실제 요구사항, 분석, 설계, 구현 규칙은 `.ai/` 아래 문서를 기준으로 확인합니다.
- 사용자의 요구사항은 항상 `.ai/requirements/`에 저장된 문서를 기준으로 추적합니다.
- 코드 생성, 수정, 설계, 문서화 작업 전에는 반드시 `.ai/rules/`의 규칙을 확인합니다.
- `.ai/` 문서와 실제 코드가 충돌하는 경우, AI 에이전트는 임의로 판단하지 않고 충돌 내용을 사용자에게 보고합니다.

---

## 2. `.ai/` 디렉터리 안내

아래 항목은 `.ai/` 아래의 1-depth 폴더 또는 파일만 설명합니다.

### `.ai/agents/`

AI 에이전트별 역할, 작업 방식, 에이전트 운영 지침을 저장하는 공간입니다.

- 특정 에이전트의 책임 범위
- 에이전트별 프롬프트 또는 동작 지침
- 역할 분담 기준

AI 에이전트의 역할이나 작업 방식이 불명확하면 이 디렉터리의 문서를 확인합니다.

---

### `.ai/docs/`

프로젝트 문서, 분석 문서, 설계 문서, API 문서 등 산출물을 저장하는 공간입니다.

- 프로젝트 개요
- 분석 문서
- 설계 문서
- API 설계 문서
- 테스트 계획 문서
- 작업 가이드 문서

문서 기반으로 프로젝트를 이해해야 할 때 먼저 확인합니다.

---

### `.ai/plans/`

작업 계획, 실행 계획, 단계별 구현 계획을 저장하는 공간입니다.

- 기능 구현 계획
- 리팩터링 계획
- 마이그레이션 계획
- 작업 분해 내역
- 진행 상태 정리

구현 전에 무엇을 어떤 순서로 진행해야 하는지 확인할 때 사용합니다.

---

### `.ai/project/`

프로젝트 자체에 대한 메타 정보와 운영 기준을 저장하는 공간입니다.

- 프로젝트 목적
- 기술 스택
- 아키텍처 개요
- 실행 방법
- 운영 방식
- 프로젝트 범위

프로젝트의 전체 맥락을 파악해야 할 때 확인합니다.

---

### `.ai/requirements/`

사용자의 요구사항을 저장하는 공간입니다.

- 사용자가 요청한 기능
- 변경 요청
- 제약 조건
- 제외 범위
- 미확정 사항
- 요구사항 변경 이력

모든 분석 문서와 설계 문서는 이 디렉터리에 저장된 요구사항 문서를 바탕으로 작성해야 합니다.

AI 에이전트는 요구사항에 없는 기능, 정책, API, 데이터 모델을 임의로 추가하지 않습니다.  
추가가 필요하면 먼저 사용자에게 확인한 뒤 요구사항 문서를 갱신합니다.

---

### `.ai/rules/`

AI 에이전트가 반드시 따라야 하는 프로젝트 규칙을 저장하는 공간입니다.

- API 설계 규칙
- 코딩 컨벤션
- 네이밍 규칙
- 보안 규칙
- Spring/Jakarta EE 패턴
- 검증 및 테스트 기준

코드 생성, 코드 수정, API 설계, 문서 작성 전에는 항상 이 디렉터리의 규칙을 확인합니다.

---

### `.ai/scripts/`

프로젝트 작업을 보조하는 스크립트를 저장하는 공간입니다.

- 문서 생성 스크립트
- 검증 스크립트
- 자동화 스크립트
- 보조 실행 스크립트

반복 작업이나 자동화가 필요한 경우 이 디렉터리를 확인합니다.

---

## 3. AI 에이전트 작업 순서

AI 에이전트는 일반적으로 다음 순서로 작업합니다.

1. `AGENTS.md`를 읽고 프로젝트 안내 구조를 파악합니다.
2. `.ai/project/`에서 프로젝트 개요와 기술 스택을 확인합니다.
3. `.ai/requirements/`에서 사용자의 요구사항을 확인합니다.
4. `.ai/rules/`에서 작업에 필요한 규칙을 확인합니다.
5. `.ai/docs/`에서 기존 분석/설계/API 문서를 확인합니다.
6. `.ai/plans/`에서 관련 작업 계획이 있는지 확인합니다.
7. 필요한 경우 `.ai/scripts/`의 자동화 도구를 확인합니다.
8. 작업 결과가 요구사항과 규칙을 위반하지 않는지 검토합니다.

---

## 4. 요구사항 기준 작업 원칙

- 사용자의 요구사항은 항상 `.ai/requirements/`에 문서로 저장되어야 합니다.
- 분석 문서와 설계 문서는 반드시 `.ai/requirements/`의 요구사항 문서를 바탕으로 작성합니다.
- 요구사항 문서에 없는 내용을 분석/설계/구현에 반영하지 않습니다.
- 요구사항이 불명확하면 AI 에이전트는 임의로 해석하지 않고 사용자에게 확인합니다.
- 확인된 내용은 먼저 `.ai/requirements/`에 반영한 뒤 후속 문서나 코드를 작성합니다.

---

## 5. 규칙 기준 작업 원칙

- 코드 생성 또는 수정 전에는 반드시 `.ai/rules/`를 확인합니다.
- API 설계 전에는 API 규칙과 보안 규칙을 확인합니다.
- Java/Spring/Jakarta EE 코드를 작성할 때는 코딩 규칙, 네이밍 규칙, Spring 패턴 규칙을 확인합니다.
- 테스트 또는 검증이 필요한 작업은 검증 규칙을 기준으로 판단합니다.
- 규칙과 요구사항이 충돌하면 AI 에이전트는 충돌 내용을 명시하고 사용자에게 확인합니다.

---

## 6. 문서 작성 원칙

- 프로젝트 문서는 `.ai/docs/` 아래에 작성합니다.
- 작업 계획은 `.ai/plans/` 아래에 작성합니다.
- 사용자 요구사항은 `.ai/requirements/` 아래에 작성합니다.
- 프로젝트 메타 정보는 `.ai/project/` 아래에 작성합니다.
- 문서를 작성하거나 수정할 때는 기존 문서와 중복 또는 충돌이 없는지 확인합니다.

---

## 7. 주의사항

- AI 에이전트는 최종 의사결정자가 아닙니다.
- 보안, 인증/인가, 개인정보, 운영 정책과 관련된 변경은 반드시 사용자 검토가 필요합니다.
- 외부 라이브러리 추가, 아키텍처 변경, 데이터 모델 변경은 요구사항과 규칙을 모두 확인한 뒤 진행합니다.
- 확실하지 않은 내용은 추정하지 말고 질문합니다.
- 작업 결과에는 어떤 요구사항과 규칙을 기준으로 했는지 명확히 남기는 것이 좋습니다.

---

## 8. 요약

이 프로젝트에서 AI 에이전트는 다음 원칙을 따릅니다.

- `AGENTS.md`는 프로젝트 작업의 진입점입니다.
- `.ai/requirements/`는 사용자 요구사항의 기준 저장소입니다.
- `.ai/rules/`는 코드, 설계, 문서 작업의 기준 규칙 저장소입니다.
- `.ai/docs/`는 분석/설계/문서 산출물 저장소입니다.
- `.ai/plans/`는 작업 계획 저장소입니다.
- `.ai/project/`는 프로젝트 메타 정보 저장소입니다.
- `.ai/agents/`는 에이전트 운영 지침 저장소입니다.
- `.ai/scripts/`는 자동화 및 보조 스크립트 저장소입니다.
